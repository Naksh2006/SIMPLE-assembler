Assembler Implementation
------------------------------------------------------------
The assembler is implemented as a two-pass assembler.

Pass 1:
- Parses assembly source lines.
- Removes comments and whitespace.
- Identifies labels and builds the symbol table.
- Detects duplicate or invalid labels.

Pass 2:
- Generates machine code.
- Resolves operands and labels.
- Calculates branch offsets for branch instructions.
- Produces machine instructions.

Assembler Features:
- Instruction table containing opcode definitions.
- Detection of common assembly errors:
  * Duplicate labels
  * Undefined labels
  * Invalid mnemonics
  * Missing operands
  * Extra operands
  * Invalid number formats
- Generates object file (.o) in binary format.
- Generates human readable listing file (.lst).
- Generates log file (.log) with errors and warnings.
- Supports SET pseudo instruction.
- Supports DATA directive.

Listing File
------------------------------------------------------------
The assembler generates a listing file showing:

Address | Machine Code | Instruction

Example format:
00000000 00000111 ldc 5

Labels are also displayed in the listing output.

Emulator Implementation
------------------------------------------------------------
The emulator simulates the SIMPLE machine architecture.

Registers implemented:
- A (Accumulator)
- B (Register B)
- PC (Program Counter)
- SP (Stack Pointer)

Features of the emulator:
- Loads object file into memory.
- Decodes machine instructions.
- Executes instructions according to opcode.
- Supports all instructions defined in the SIMPLE ISA.
- Detects invalid memory accesses.
- Detects illegal opcodes.
- Stops execution on HALT instruction.
- Limits instruction execution to avoid infinite loops.

Additional emulator options implemented:
- -trace   : shows register state during execution
- -before  : memory dump before execution
- -after   : memory dump after execution
- -isa     : prints supported instruction set
